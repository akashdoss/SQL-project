# Retail Insights – Stock Health Report (SQL)

This project is based on a real-world business scenario where a supermarket chain, *FreshMart*, wants to improve its inventory management using data analysis. The goal is to identify products that are not selling well and those that are at risk of expiring soon.

## Overview

Retail businesses often face losses due to overstocking or poor inventory planning. This project focuses on generating a "Stock Health Report" using SQL to help make better decisions based on sales and product data.

## Objectives

* Identify products that are nearing expiration but still have high stock
* Detect "dead stock" items that have not been sold recently
* Analyze which product categories generate the most revenue

## Database Schema

The project uses three main tables:

* **Categories** → Stores product categories
* **Products** → Contains product details like stock, price, and expiry date
* **SalesTransactions** → Records sales activity for each product

## Key Queries

### 1. Expiring Soon Products

Finds products that will expire within the next 7 days and still have a stock count greater than 50.

### 2. Dead Stock Analysis

Identifies products that have not been sold in the last 60 days.

### 3. Revenue Contribution

Calculates total revenue generated by each category in the last month using aggregation.

## Technologies Used

* SQL (MySQL syntax)
* Basic concepts like JOIN, GROUP BY, and aggregate functions

## How to Run

1. Open any SQL environment (MySQL Workbench / DB Browser / etc.)
2. Run the provided `.sql` script
3. Execute the queries to view the results
